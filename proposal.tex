\Keywords{fraud detection, device fingerprinting, device intelligence, risk scoring, payment service provider, SIM swap, digital payments, transaction monitoring, Nepal, web SDK}

\begin{ncitabstract}
Nepal's digital payments are growing fast, but many payment providers still rely on simple fixed rules that miss organised fraud. SIM swap attacks increased sharply in recent years~\cite{fiu2024}, and banks face stronger compliance checks. This project proposes \textbf{Flint Secure}: a fraud and device-intelligence service that Nepali providers can host themselves. The team will build a Go API and background workers with Redis, NATS, and PostgreSQL. Two main services are planned: a \textbf{device identification API} using a three-step fingerprint match, and a \textbf{transaction scoring API} that combines seven risk factors with SIM-swap safety rules. A web SDK will collect device signals on the client side. The system should return fraud decisions (ALLOW, FLAG, or BLOCK) in under 50\,ms for most requests and handle about 3,200 requests per second. Testing will use k6 load scripts and about 168 automated test cases, aiming for at least 92\% passing.
\end{ncitabstract}
